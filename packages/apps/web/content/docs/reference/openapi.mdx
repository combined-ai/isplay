The API server publishes OpenAPI at:

```text
GET /openapi.json
```

The document is generated by Hono zod-openapi route definitions in `@isplay/server`.

## Local Fetch

```bash
npx isplay start
curl http://127.0.0.1:7373/openapi.json
```

## What It Covers

The generated document covers registered HTTP routes, request bodies, route parameters, query parameters, response schemas, and structured error responses for route definitions.

## What Still Needs Human Docs

OpenAPI does not explain:

- Evidence strength.
- Replay policy tradeoffs.
- Fixture provenance.
- Low-N experiment caveats.
- Adapter capability differences.
- Current executor limitations.

Use OpenAPI for exact HTTP shape. Use the concept and guide pages for interpretation.
